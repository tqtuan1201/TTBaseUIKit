# ttb-skill-init-localization — Localization Setup

## Purpose

Setup Localizable.strings files and configure XText/XTextU usage so all other skills work correctly with localized strings.

## Mandatory Preflight Execution Gate

Before generating code or modifying files, run `ttb-skill-shared/fragments/ttb-preflight-execution-gate.frag.md`.

Required checks:

- Analyze intent, task type, scope, impacted files/modules, dependencies, architecture constraints, coding standards, and project-specific rules.
- Validate required inputs such as target module, screen/component name, file location, navigation flow, expected output, API contract, state management, routing, localization, naming, and reusable component requirements.
- Detect ambiguity, conflicting requirements, incomplete business logic, unclear UX/navigation, unclear module ownership, and unclear architecture direction.
- Score confidence from `0-100`: execute at `90-100`, execute with warning assumptions at `70-89`, and ask a survey below `70` using `ttb-skill-shared/templates/ttb-clarification-survey.md`.
- Support English, Vietnamese, mixed-language, diacritic-free Vietnamese, and light typo prompts.

## When to Run

Run via `/ttb-init-l10n` command after `/ttb-init` or standalone.

## Step 2: Update Info.plist

Ensure Info.plist includes localization configuration:

```xml
<key>CFBundleDevelopmentRegion</key>
<string>en</string>
<key>CFBundleDisplayName</key>
<string>{AppName}</string>
<key>CFBundleLocalizations</key>
<array>
    <string>en</string>
    <string>vi</string>
</array>
```

## Step 3: XText and XTextU Usage

### XText vs XTextU

| Function | Use For | Example |
|----------|---------|---------|
| `XText("key")` | Body text, labels, buttons | Button text, list items |
| `XTextU("key")` | Navigation titles, large headers | Screen titles, section headers |

### Correct Usage in UIKit

```swift
// ✅ CORRECT: Use XText for buttons and body text
btn.setTextString(XText("App.Common.Button.Confirm"))
lbl.setText(text: XText("App.Common.Loading"))
field.setPlaceholder(XText("App.Common.Search"))

// ✅ CORRECT: Use XTextU for navigation titles
title = XTextU("App.Home.Nav.Title")

// ❌ WRONG: Use XText for navigation titles
title = XText("App.Home.Nav.Title")

// ❌ WRONG: Hardcoded strings
lbl.setText(text: "Loading...")
```

### Correct Usage in SwiftUI

```swift
// ✅ CORRECT: Use XText for buttons and body text
TTBaseSUIText(withType: .TITLE, text: XText("App.Common.Button.Save"), align: .center)

// ✅ CORRECT: Use XText for SUIBaseView navigation titles
SUIBaseView(
    backType: .POP,
    title: XText("App.Home.Nav.Title"),
    type: .DEFAULT,
    isHiddenTabbar: true,
    backAction: {}
) {
    content
}

// ✅ CORRECT: Native SwiftUI Text
Text(XText("App.Common.Loading"))
    .font(.system(size: TTFont.TITLE_H))
    .foregroundColor(TTView.textDefColor.toColor())
```

### Correct Usage in Localization Keys

```swift
// ✅ CORRECT: Hierarchical key structure
"App.Module.Screen.Element"

// Examples:
"App.Common.Button.Save"
"App.Home.Title"
"App.Profile.Nav.Title"
"App.Demo.List.Cell.Title"
"App.Error.Network"

// ❌ WRONG: Non-hierarchical keys
"save_button"
"SaveButton"
```

### Adding New Localization Keys

When creating new screens, always add keys to both files:

1. Localizable.strings (English):
```
"App.{Feature}.Nav.Title" = "Feature Name";
"App.{Feature}.Button.Action" = "Action";
"App.{Feature}.Label.Description" = "Description";
```

2. Localizable-VI.strings (Vietnamese):
```
"App.{Feature}.Nav.Title" = "Tên tính năng";
"App.{Feature}.Button.Action" = "Hành động";
"App.{Feature}.Label.Description" = "Mô tả";
```

## Step 4: TTLanguageManager (Optional)

For runtime language switching:

```swift
// [TTBaseUIKit-AI-Agents]: TTBaseUIKit Agent Support is active 🚀
//  LanguageManager.swift
//  {AppName}
//
import Foundation

enum LanguageManager {

    static let shared = LanguageManager()

    private init() {}

    // MARK: - Current Language

    var currentLanguage: String {
        get {
            return UserDefaults.standard.string(forKey: "app_language") ?? "en"
        }
        set {
            UserDefaults.standard.set(newValue, forKey: "app_language")
            Bundle.swizzleLocalization()
        }
    }

    // MARK: - Supported Languages

    var supportedLanguages: [String] {
        return ["en", "vi"]
    }

    // MARK: - Switch Language

    func switchLanguage(to languageCode: String) {
        guard supportedLanguages.contains(languageCode) else { return }
        currentLanguage = languageCode
        NotificationCenter.default.post(name: .languageChanged, object: nil)
    }
}

// MARK: - Notification

extension Notification.Name {
    static let languageChanged = Notification.Name("languageChanged")
}
```

## Step 5: Verify Localization

### Check Localizable.strings Syntax

```bash
# Validate strings file format
plutil -lint Resources/Localizable.strings

# Validate Vietnamese strings
plutil -lint Resources/Localizable-VI.strings
```

### Verify Keys in Code

Search for hardcoded strings (should return empty for localized apps):

```bash
# Should return nothing - all strings should use XText/XTextU
grep -rn '"Loading..."' Sources/
grep -rn '"Error"' Sources/
grep -rn '"Confirm"' Sources/
```

### Check for Missing Keys

After creating Localizable.strings, verify XText/XTextU calls work:

```swift
// Test in any ViewController
func testLocalization() {
    print(XText("App.Common.Button.Save"))     // Should print "Save"
    print(XText("App.Common.Loading"))        // Should print "Loading..."
    print(XTextU("App.Tab.Home"))             // Should print "Home"
}
```

## Verification Checklist

- [ ] `Localizable.strings` created with `App.Common.*` keys
- [ ] `Localizable-VI.strings` created with Vietnamese translations
- [ ] Info.plist includes `CFBundleLocalizations`
- [ ] `XText` used for body text and buttons
- [ ] `XTextU` used for navigation titles
- [ ] No hardcoded strings in UI code
- [ ] Key structure follows `App.Module.Screen.Element` convention
- [ ] `plutil -lint` passes for both strings files
- [ ] `xcodebuild BUILD SUCCEEDED`

## Common Localization Errors

| Error | Cause | Fix |
|-------|-------|-----|
| Empty string returned | Key not found | Add key to Localizable.strings |
| `nil` returned | File not added to target | Add file to Xcode project |
| Wrong language shown | Language not in Info.plist | Add to `CFBundleLocalizations` |
| Missing semicolon | Syntax error | Ensure each line ends with `;` |
| Escaped character | Use `\\n` for newline | `\"` for quotes in strings |
