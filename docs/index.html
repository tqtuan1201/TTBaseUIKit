Why TTBaseUIKit

Designed to remove friction between UIKit and SwiftUI while keeping teams aligned.

One system for both UI stacks

Share consistent patterns across UIKit and SwiftUI without re-inventing components.

Faster delivery, fewer regressions

Reusable base views and cells reduce duplicate work and keep UI predictable.

Configurable design tokens

Centralized fonts, sizes, and styles through BaseConfig keep branding uniform.

Documentation standards inspired by top design systems

TTBaseUIKit documentation follows proven guidance: include usage, code, and accessibility notes per component, and add dos and don’ts where behavior matters. These practices appear consistently across mature documentation systems.

Installation

Use Swift Package Manager to integrate TTBaseUIKit into your project.

Swift Package Manager

dependencies: [
    .package(url: "https://github.com/your-org/TTBaseUIKit.git", from: "1.0.0")
]

Add the package, then import TTBaseUIKit in your modules.

Requirements

  • iOS 13+
  • Swift 5.7+
  • Xcode 15+

Usage

Quick examples for UIKit and SwiftUI.

UIKit Example

import TTBaseUIKit

let view = BaseUIView()
view.translatesAutoresizingMaskIntoConstraints = false
parentView.addSubview(view)

SwiftUI Example

import TTBaseUIKit
import SwiftUI

struct DemoView: View {
    var body: some View {
        BaseSUIView()
            .padding()
    }
}
